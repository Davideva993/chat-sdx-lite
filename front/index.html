    Key Principles

    • No unnecessary user data (phone, email, account...)
    • Sensitive data and keys never leaves the browser in cleartext
    • Security over comfort: session aborts instantly if compromise is suspected
    • Server stores only the last 6 encrypted messages (real or fake), temporarily
    • No cookies, no persistent storage, and no global variables — reloading the page wipes everything. The room and all browser-side data auto‑clear after 6 hours of real inactivity.
    • Every message encrypted with a unique key, computed locally, and unknown to the server
    • Knowing one message key is not sufficient to derive previous or next keys
    • Each message is padded and when the chat stops an automatic fake-messages system starts
    • It’s suggested to host it through Tor

    Host a chat: next steps:

    1. Choose the secret words
    2. Click "host this chat"
    3. Share the roomName and secret words with the other user via a temporary secure channel
    4. Wait for the other user to join

    If the page reloads, the room is lost.

    Join a chat: next steps:

    1. Use a temporary secure channel to obtain the room code and secret words
    2. Enter the information and click “Join this chat”

    If the page reloads, the room is lost.
